# Project Vision — 為什麼做這個

Last updated: 2026-07-11

## 一句話
讓**多個 AI（各自吃不同訂閱）用共享記憶接力、共同開發同一個專案**。

## 真正的意義（不要搞錯重點）
這個專案的心臟**不是「輪替省額度」**，是**「協作」**：
- 輪替（額度到了換手）只是**觸發交接的時機**。
- 真正讓每次交接是「延續」而不是「失憶重來」的，是那份**共享記憶**（`docs/`）。
- 沒有共享記憶，換手 = 每次換一個不知道前因後果的人進來 → 「接手的 AI 根本不知道要做什麼」→ 整套失去意義。

## 共享記憶怎麼成立（關鍵洞見）
四家 CLI 不能共享**對話上下文視窗**（不同模型/廠商，技術上不可能）。
它們共享的是 **File-resident memory**：狀態存在**文件**裡、不在對話歷史裡 →
才能跨 agent、跨工具無痛切換。**那份 `docs/` 紀錄就是共享大腦，四個 CLI 是可替換的手。**
（此設計參考 Agent-LV.MAX Record System：https://github.com/donbeeshyvt-jpg/Agent-LV.MAX-record-system）

## 成功長什麼樣
1. claude 額度沒了 → codex 接手，讀 `docs/` 就知道剛做到哪、接下來幹嘛，繼續同一個任務。
2. 使用者在**本地 web 控制台**就能：看額度、派工、對話（背後共享記憶）、跑技能。
3. 對話跨越換手仍連續——這句 claude 答、下句 codex 答，codex 讀得到前面全部。

## 非目標 / 邊界
- 不是把訂閱轉售 / 分享 / 對外當服務（那會違反各家 ToS）。個人自用、適量。
- 額度「檢查」一律讀本地 log，不打遠端 API（零帳號風險）。
