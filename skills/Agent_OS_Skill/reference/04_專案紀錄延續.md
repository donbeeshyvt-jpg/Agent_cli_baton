# 04 ｜ 專案紀錄延續（Context Intake：偵測「多 agent 協作紀錄系統」並延續思考）

> **這是純追加的「軟性」能力，不改動 12 角色、不改結構。**
> 「軟性」＝全部是 prompt 指示，讓 AI **主動去找、去讀** 專案的 `docs/`；**不安裝、也不依賴任何 hook**。讀不到就優雅降級，絕不卡死、絕不幻覺。
> 目的：當專案是用 **「Agent 協作紀錄與規劃專案系統」**（唯一入口 `AGENTS.md` + `docs/` 外接記憶）建的，Agent OS 啟動時**嘗試**先讀它的 `docs/`（尤其使用者對話紀錄），再進主流程——達成三件事：
> 1. **延續思考**：讀 `docs/CONVERSATION_LOG.md` 的 Active Summary 與近期使用者訊息，接上上次的討論，不重問已知的事。
> 2. **跟進程式進度**：讀 `docs/HANDOFF.md` / `docs/TASKS.md`，知道做到哪、下一個安全任務是什麼。
> 3. **更準理解真意**：讀 `docs/PROJECT_VISION.md` / `docs/INTENT_TRACE.md`，把初衷與意圖鏈餵給意圖解碼者。

---

## A. 偵測（fingerprint）— 這專案是不是用該系統建的？

**最小判定（採 OR，避免漏判）**：專案根目錄有 `AGENTS.md`，且 `docs/00_AI_CONTEXT_INDEX.md` 存在，且（`docs/CONVERSATION_LOG.md` **或** `docs/HANDOFF.md` 其一存在）→ **命中**。
（這些是該系統的專屬檔名指紋；一般專案不會剛好同時有這幾個。指紋命中只代表「值得去讀」，不代表內容一定齊全——見 §F。）

## B. 命中後的讀取順序（省 token、**不要掃全庫**）

按這個順序讀（沿用該系統自己在 `HANDOFF.md` §Notes for Next Agent 規定的接手法）。**任一檔不存在就跳過、不報錯**（見 §F）：

1. `docs/00_AI_CONTEXT_INDEX.md` — 專案目的、現況、下一步、技術棧（總覽，先讀這個定位）
2. **`docs/CONVERSATION_LOG.md`** — ⭐核心：先讀 **Active Summary**（最新方向／最近修正／未決問題／最近批准），再掃**最近幾則帶時間戳的使用者訊息**＝「延續使用者的思考」。⚠️ 這兩者**可能為空**（該系統的 auto-hook 尚未安裝時就是空的）——見 §F，**空就當「無歷史」，絕不幻覺**。
3. `docs/HANDOFF.md` — 目前狀態、最後完成任務、**下一個安全任務**、禁改清單（跟進程式進度）；若 `docs/loops/LOOP_PORTFOLIO.md` 存在，順帶讀艦隊總表（現役迴圈、pause-all、Human Inbox）——NEEDS_LOOP 任務必讀
4. `docs/GOLDEN_RULES.md` — 專案硬規則／禁改清單（帶進 ② 任務架構師、⑦ 最小變更實作者當硬約束）
5. `docs/REQUEST_LOG.md` — 最新 **Approvals / Corrections**（已同意什麼、已被更正什麼）
6. `docs/PROJECT_VISION.md` + `docs/INTENT_TRACE.md` — 初衷與意圖鏈（更準對齊使用者真意）
7. `docs/TASKS.md`（或 `docs/features/NNN-*/tasks.md`）+ `docs/CODE_INDEX.md` — 任務進度 + 碼庫地圖（**只讀需要的 1–3 個檔**）
8. 可選（只在相關且存在時才讀）：`docs/REQUIREMENTS.md`（憲章級約束）、`docs/DECISION_LOG.md`（已定案決策，避免重議）

> **鐵律**：不要掃描整個程式碼庫——這套紀錄系統的核心價值就是省 token，用 `docs/` 當地圖。

## C. 讀完怎麼接進 12 角色管線（不改角色，只是餵 context）

**分工先講清楚**：ⓠ／本檔只負責「把 `docs/` 讀成既有 context ＋ 產一句現況摘要」；**真正的意圖解碼（XY 檢查、成功標準、flags）仍由 ① 意圖解碼者執行**。docs context 是餵給 ① 的燃料，不取代 ①（① 的「不吃角色產物」指的是不吃下游角色的產物，`docs/` 是專案既有資料、不算角色產物）。

- `CONVERSATION_LOG` 的 Active Summary + `PROJECT_VISION`/`INTENT_TRACE` → 讓 ① **不重問已知的事**，直接對齊真意（更準）。
- `HANDOFF` 的「下一個安全任務」+ `TASKS` → 若使用者這次沒明說要做什麼，**預設接續 HANDOFF 的下一步**（延續，而非從零）。
- `GOLDEN_RULES` / `HANDOFF` 的「禁改清單」→ 當硬約束帶進 ② 任務架構師與 ⑦ 最小變更實作者。
- **先產一句「現況摘要 + 我理解你接下來想…」給使用者確認方向，再進主流程。**

## D. 讀取是「軟性」的；回寫是紀錄系統「自己的機制」（切乾淨）

- **本 skill 對 `docs/` 只做「讀取」，讀取永不依賴、也不要求安裝任何 hook。** 偵測與讀取都是 prompt 指示的軟性行為；hook 沒裝、doc 沒有，都只是降級，不是錯誤。
- 下面的**回寫規矩是紀錄系統自己的機制**，只有當你恰好在該 repo 內、且要 `git commit` 時才會遇到，跟本 skill 的讀取無關：
  - 該系統*希望* agent 每回合結尾更新 `docs/CONVERSATION_LOG.md` 的 Active Summary、動 `src/` 就同步 `docs/CODE_INDEX.md`、非小改走 `APPROVED-NNN` 批准。**以該專案的 `AGENTS.md`（及 `docs/` 內的 `GOLDEN_RULES.md`／`REQUIREMENTS.md`，若存在）為準**。
  - 它的 `pre-commit` hook **只在該 repo 已執行過 `install-hooks` 後才存在，且只在你 `git commit` 時作用**——那是它的 enforcement，**不是本 skill 的依賴**。你若沒在裡面 commit、或 hook 沒裝，本 skill 的讀取與延續照常運作。
- 一句話：**本 skill 只「配合」它、不「覆寫」也不「依賴」它。**

## E. 沒偵測到時（指紋完全未命中）

退回**通用情境攝取**：讀 `AGENTS.md` / `CLAUDE.md` / `README.md` + 回顧本對話已知事實，再進主流程。

## F. 部分缺失／空紀錄的優雅降級（重要）

指紋命中 **≠** 內容一定齊全。最常見：該系統的 auto-hook 尚未 `install`（所以 `CONVERSATION_LOG.md` 只有 scaffold 樣板、Active Summary 全是 `(none yet)`），或個別 doc 不存在。處置一律「**讀到什麼算什麼**」：

- **某 doc 不存在** → 跳過該檔，用還在的檔繼續，不報錯、不中止。
- **`CONVERSATION_LOG` 的 Active Summary 為空／全 `(none yet)`／無帶時間戳訊息** → 視為「**無歷史對話**」，**嚴禁杜撰過往討論或批准**；改以 `HANDOFF`／`PROJECT_VISION`／`README` ＋ 本次對話對齊，並在「現況摘要」明標「**無歷史對話紀錄，將以現況與你本次輸入為準**」。
- **完全讀不到 `docs/`** → 退回 §E 的通用攝取。

> 全程原則：**讀得到多少用多少；缺失只降級、不中止、不幻覺。**
